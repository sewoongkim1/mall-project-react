## 보안 감사 및 수정

### 보안 감사 수행 (OWASP Top 10 기준)
- 총 20개 취약점 식별: Critical 3개, High 4개, Medium 7개, Low 4개

### Critical 수정 완료

| 취약점 | 수정 내용 |
|--------|----------|
| JWT 시크릿 취약 | 256-bit 랜덤 시크릿으로 교체 (crypto.randomBytes) |
| 소셜 로그인 URL에 토큰 노출 | URL 파라미터 제거, httpOnly 쿠키로만 전달 |
| 결제 금액 조작 가능 | 서버에서 DB 가격 조회 후 클라이언트 가격과 대조 검증 |

### High 수정 완료

| 취약점 | 수정 내용 |
|--------|----------|
| 관리자 자기 역할 변경 | 자기 자신 ID 비교 후 403 에러 반환 |
| NoSQL injection (status/role 필터) | 유효 enum 값 whitelist 검증 후 MongoDB 쿼리에 전달 |

### 미수정 (향후 개선)

| 심각도 | 항목 |
|--------|------|
| High | localStorage 토큰 저장 (httpOnly 쿠키 전환 권장) |
| High | OAuth state 파라미터 미사용 (CSRF 방어) |
| Medium | 비밀번호 복잡도 강화, CSRF 토큰, 프롬프트 인젝션 방어 |
| Low | 토큰 블랙리스트 (로그아웃 시 즉시 무효화), 감사 로그 |
